Claude Shell Help

Getting Started

Claude Shell is a management tool for sharing Claude Code CLI sessions remotely. It discovers running sessions, creates WebSocket proxy bridges, and configures tunnel providers to expose them for remote access.

Important: Claude Shell is a management console only. All interaction with Claude sessions happens through the remote web client that connects via the tunnel.

Interface Overview

Sessions Tab

Displays all discovered Claude Code sessions running on your machine. Sessions are scanned from ~/.claude/sessions/ every 3 seconds.

ColumnDescription
PIDProcess ID of the Claude CLI session
NameSession display name (if set)
CWDWorking directory of the session
StartedTime the session was started
StatusWhether the process is alive or dead

Select a session and click Start Proxy to create a WebSocket bridge.

Proxies Tab

Shows running proxy bridges. Each proxy spawns a claude --print process and exposes it via WebSocket.

  • Start Tunnel — Expose the proxy via a tunnel provider
  • Stop Proxy — Terminate the proxy and claude process
  • Copy WS URL — Copy the local WebSocket URL to clipboard

Tunnels Tab

Configure and manage tunnel providers to expose proxies to the internet.

Supported providers:

  • ngrok — ngrok http <port>
  • Microsoft Dev Tunnels — devtunnel host -p <port>
  • Cloudflare Tunnel — cloudflared tunnel --url http://localhost:<port>

Click Copy Connection String to get the shareable URL with auth token.

Logs Tab

Color-coded log output for debugging. Shows session discovery, proxy events, tunnel status, and errors.

Remote Access

Once a tunnel is running, open the public URL in a browser. The remote web client allows full interactive control of the Claude session:

  • Send prompts and receive streaming responses
  • Approve or reject tool calls
  • Abort running responses
  • View conversation history

Security

  • Each proxy generates a 256-bit random auth token
  • Tunnel providers handle TLS encryption
  • Only one controller per session (additional connections become viewers)
  • Never share connection strings with untrusted parties

Troubleshooting

IssueSolution
No sessions foundEnsure Claude Code is running (claude in terminal)
Proxy fails to startCheck that claude is in your PATH
No tunnel providersInstall ngrok, devtunnel, or cloudflared
Tunnel URL not appearingCheck Logs tab for tunnel process output

System Requirements

  • macOS 13.0 (Ventura) or later
  • Claude Code CLI installed
  • At least one tunnel provider (ngrok, devtunnel, or cloudflared)
